Simulation Command Center

To demonstrate **SentinelMesh's Autonomous Response Capabilities**, you can launch telemetry simulations. The ML models in the Python engine will detect anomalies and report them via gRPC, which triggers the incident correlation and mitigation pipelines.

How to run the simulator:

  1. Open a terminal in the folder: c:\Users\bened\Documents\JAVA PROJECT\sentinelmesh-ai
  2. Run the command: python simulator.py
  3. Select one of the simulated attack modes in the CLI prompt!
Mode 1: Normal Workload

Generates baseline metric patterns across random hosts. Use to see steady state monitoring.

Mode 2: Volumetric DDoS

Floods host server-us-01. Triggers MOCK_RATE_LIMIT_IP mitigation.

Mode 3: Authentication Abuse

Floods host server-ap-01 with failed logins. Triggers MOCK_BLOCK_IP mitigation.

Mode 4: System Memory Leak

Climbs RAM on server-eu-02. Triggers MOCK_SCALE_UP mitigation.

Mode 5: CPU Saturation

Spikes CPU load on server-us-03. Triggers MOCK_SCALE_UP mitigation.
